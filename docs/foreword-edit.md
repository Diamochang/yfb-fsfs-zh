距离本书的英文原版问世，已经过去十余年了。在这十年期间，自由软件运动仍旧得到进一步的发展，但在发展过程中也遇到了不少挑战。就在北京 GNU/Linux 用户组（本书后称“BLUG”）最后一次修订本书的第一个中译本三个月后，中国香港的“修例风波”大规模爆发，这引发了自由软件开发者与政治之间关系的思考和讨论（典型如 Notepad++ 开发者曾计划破坏不支持其政治立场用户的使用体验，引发社区反对）。与此同时，流媒体平台的数字限制管理技术越发受到自由软件基金会的严重关切，其早在 2006 年组织的设计性缺陷（Defective by Design）运动也开始越发活跃。在世界上，新冠疫情、地缘政治加剧和霸权主义的抬头也使得自由和开源软件获得了更多机遇，它无国界和给予用户充分自由的属性受到越来越多人，尤其是发展中国家人民和饱受剥削的资本主义发达国家人民的关注。自由和开源软件站在了时代变局的风口上。

在这种形势下，作为世界上最大发展中国家的中国，参与自由和开源软件开发的开发者也在变多。但由于自由软件运动的精神尚未得到更加广泛的传播，我国自由软件运动在这十年的发展中，既有 Apache Kylin、RocketMQ 等优秀软件，深度求索 R1 和 V3、阿里 Qwen 系列等真正实现社区和商业“双赢”的开源人工智能模型，也有 BootCSS 和 LNMP 供应链投毒、贵州某公司“收买” Alist 并试图投毒等恶例。Linux 中国的停运，实际上减慢了自由软件运动精神的传播速度，而这本书作为反映自由软件运动基本思想的著作之一，倘若能广泛传播，可以使大家更深层次的领会自由软件运动精神，促进我国的自由软件运动健康、茁壮发展。本着这样的目的，我根据 GNU 网站中文翻译者团队（本书后称“GNU CTT”）在 2019 年之后对网站上文章翻译的更新，对 BLUG 中译本做了编辑，形成现在大家所读到的这个编辑本，作为“自由和开源软件先锋文库”的一部分。

本次编辑秉持尊重原著的原则，不大量增删内容，不调整收入文章的数量及顺序，仅对原译文中部分字词句做出修改，以使整部译著读起来更顺畅，符合传播需要。《自由软件之歌》的中文乐谱也加上了词曲及歌词译者信息，乐谱文件可以使用 MuseScore 的最新版本（撰写时是 4.6.5）直接编辑，不需要另存为新文件。

如果对本书的编辑有什么问题和建议，欢迎在编辑本的公开源码仓库 <https://github.com/Diamochang/yfb-fsfs-zh> 中提交 Issues 或 Pull Requests。最后，我谨向参与翻译本书的 BLUG 成员以及在 GNU 网站上持续维护线上文章翻译的 GNU CTT 成员们表示最诚挚的敬意，希望他们继续为中国的自由软件运动发光发热，与我们这群更年轻的开发者一起将具有无限价值的数字公共产品奉献给下一代。
